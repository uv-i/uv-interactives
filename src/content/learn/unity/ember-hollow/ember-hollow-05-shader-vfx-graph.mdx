Open **Packages ‚Üí Ember Hollow ‚Üí Scripts ‚Üí VFX ‚Üí `CrumbleWall.cs`**, and the
`Dissolve` Shader Graph you built in Chapter 0.

## The problem ‚Äî crumbling from the edges, not shrinking

The north wall needs to visibly disintegrate as the ruin collapses. Scaling the
mesh to zero or fading its alpha uniformly reads as "shrinking" or "fading," not
"crumbling" ‚Äî nothing looks like it's breaking apart from the edges inward.

## The fix ‚Äî a shader property animated at runtime

The `Dissolve` Shader Graph exposes a float, `_DissolveAmount`, feeding a Noise node
into an Alpha Clip Threshold ‚Äî pixels below the threshold get clipped entirely,
and as the noise pattern is uniform-random, the clipped region looks like
disintegration rather than a clean wipe.

`CrumbleWall.cs` animates that property per-renderer:

```csharp
private IEnumerator DissolveRoutine()
{
    if (debrisBurst != null) debrisBurst.Play();

    float t = 0f;
    while (t < dissolveDuration)
    {
        t += Time.deltaTime;
        float amount = t / dissolveDuration;

        _renderer.GetPropertyBlock(_propBlock);
        _propBlock.SetFloat(dissolveProperty, amount);
        _renderer.SetPropertyBlock(_propBlock);

        yield return null;
    }
    gameObject.SetActive(false);
}
```

## Why MaterialPropertyBlock, not `sharedMaterial`

```csharp
private MaterialPropertyBlock _propBlock;
```

`MaterialPropertyBlock` overrides a property **per-renderer** without touching the
shared `Material` asset. If `CrumbleWall.cs` instead did
`_renderer.sharedMaterial.SetFloat(...)`, every object using that material ‚Äî
every wall built from the same shader ‚Äî would dissolve together, because they'd
all be reading and writing the same asset.

## The counterfactual

Swap `SetPropertyBlock` for directly editing `_renderer.sharedMaterial` and add a
second `CrumbleWall` using the same `Dissolve` material. Collapse one wall ‚Äî both
dissolve, because they share the one material instance you just mutated.

## Try it ‚Äî break it on purpose

Temporarily rename `dissolveProperty` in the Inspector to a property that doesn't
exist on the shader (e.g. `_Typo`). Play and trigger the collapse ‚Äî nothing errors,
and the wall's debris still bursts, but the mesh never visually dissolves.
`SetFloat` on a non-existent property silently no-ops rather than throwing.

## Interview corner

**What does Alpha Clip actually do to a pixel?** It discards the pixel entirely if
its computed alpha falls below the threshold ‚Äî the pixel isn't drawn transparent,
it isn't drawn at all, which is what allows a mesh to appear to have literal holes
rather than just fading.

**Why is the debris `ParticleSystem` burst triggered from code instead of an
Animation Event on the dissolve?** `CrumbleWall.Collapse()` is called by
`CollapseTimer` at a moment decided by game logic, not by an animation clip's
timeline ‚Äî there's no clip driving this at all, so the burst has to be triggered
from the same script that starts the dissolve coroutine.

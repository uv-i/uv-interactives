A scene is a level. This module makes yours *look* like one ‚Äî then teaches you to
move between several without losing your managers on the way.

## Terrain ‚Äî digital clay

Open **`Module04_SceneBuilding/Scenes/01_Terrain`**. Create a Terrain
(**GameObject ‚Üí 3D Object ‚Üí Terrain**) and just‚Ä¶ sculpt for ten minutes. Raise/Lower,
Smooth, then **Paint Texture** with a Terrain Layer (grass, rock). The Terrain
Collider comes free ‚Äî characters walk on whatever you sculpt.

Then light it: a **Directional Light** is the sun, and only its *rotation* matters ‚Äî
position is irrelevant for a "infinitely far away" light. Rotate for sun angle;
recolour for mood. Deep orange = sunset, cold blue = moonlight. Two clicks, entire
scene transformed.

## Post-processing ‚Äî the gasp demo

Open **`02_PostProcessing`**. In URP: add a **Volume** to the scene, create a
Profile, add overrides ‚Äî **Bloom** (bright things glow), **Color Adjustments**
(saturation, contrast), **Vignette** (darkened edges).

Do the before/after toggle. This is the single most dramatic visual moment of the
course: the same grey scene becomes *cinematic* with three checkboxes. Then push it:
saturation ‚àí100 for noir, blue shadows + orange highlights for the Hollywood look.
Same level, entirely different film.

## Scene management ‚Äî moving between levels

Open **`03_Transitions`**. Three tools from `SceneLoader.cs` and `FadeTransition.cs`:

```csharp
SceneManager.LoadScene("Level2");            // instant switch (hitches on big scenes)
SceneManager.LoadSceneAsync("Level2");       // loads in background; progress 0‚Üí1
```

Async loading is how loading bars exist: read `operation.progress` while the current
scene keeps rendering. And the rule you already hit in Chapter 0: **a scene must be
in File ‚Üí Build Settings or loading it silently fails.**

## DontDestroyOnLoad ‚Äî the survivors

Everything in a scene dies when the next scene loads. Except:

```csharp
DontDestroyOnLoad(gameObject);   // this object now survives ALL scene loads
```

`PersistentManager.cs` uses it with a singleton guard ‚Äî the pattern for audio
managers (music that doesn't restart at every level) and game state (score that
survives the transition). The guard matters: load the first scene twice and you'd
have two managers without it. Also the warning that comes with the power: everything
persistent is effectively a global ‚Äî use it for the few true managers, nothing else.

## Try it

Build the fade: hook `FadeTransition` before the scene load, so the screen dips to
black, loads, and fades back in. Every polished game does this; yours does now too.

## Interview corner

**LoadScene vs LoadSceneAsync?** Sync freezes the frame until loaded; async loads in
the background exposing progress ‚Äî required for loading screens.

**Risks of DontDestroyOnLoad?** Duplicates on scene reload (guard with a singleton
check) and hidden global state ‚Äî keep the persistent set tiny.
